Detailed Description

Encapsulates a event function with priority-based execution ordering.

In the ECS architecture, events are functions that operate on entities with specific component sets. This class wraps such functions and provides priority ordering to control execution sequence during Registry::run_events().

events are stored sorted by priority (higher values execute first), allowing fine-grained control over update order. For example, input events might run before physics, which runs before rendering.

Template Parameters
ArgsArgument types passed to the event function (typically Registry& and component arrays)
Note
events are compared using operator< where higher priority values sort first
