72 // EVENT SYSTEM
73 // ============================================================================
74 //
75 // Provides type-safe event handling with multiple emission formats.
76 //
77 // Events are user-defined types that trigger registered handler functions.
78 // The system supports:
79 // - Type-safe handlers with automatic parameter binding
80 // - JSON-based event emission for network/config integration
81 // - Binary serialization for efficient network transmission
82 // - Handler registration/unregistration with unique IDs
83 //
84 // Handler Lifecycle:
85 // 1. Register handler with on() - returns unique ID
86 // 2. Handlers execute when emit() is called
87 // 3. Unregister with off() (specific) or off_all() (all handlers)
88 //
89 // Emission Formats:
90 // - emit(Event{...}) - Direct type-safe emission
91 // - emit<Event>(json) - JSON deserialization
92 // - emit<Event>(bytes, length) - Binary deserialization
93 //
94 // Network Integration:
95 // Events can be serialized to JSON or binary format for network transmission,
